“言外之意 Subtext”项目流程版
它的核心链路是：
混乱反馈 → 需求萃取 → 下一步行动建议 → 方向确认小样 → 客户回复话术
最终帮助广告团队减少误解、减少返工、提高下一轮方案通过率。
项目名：“言外之意 Subtext”｜Client Feedback to Action Agent
一、项目定位
「言外之意 Subtext」是一个面向广告营销团队的客户反馈工作流 Agent。
它解决的问题是：广告项目中，客户反馈往往是模糊、感受型、非结构化的，比如“再高级一点”“不要太硬广”“品牌露出不够”“达人感觉有点散”。团队需要反复猜测客户真实诉求，并将其转化为策划、媒介、设计、视频、AE 等岗位可执行的修改动作。
本项目希望通过 AI Agent，将客户的抽象反馈转化为清晰的「行动卡」，帮助团队减少理解偏差、降低返工成本、提高下一轮方案通过率。

---
二、核心痛点
广告项目真实工作流通常是：
初稿给客户看 → 客户提出模糊反馈 → 内部理解和拆解 → 修改一版 → 再给客户看 → 再反馈 → 再修改。
痛点不在于客户没有反馈，而在于客户反馈往往很难直接执行。
例如客户说：
“整体视觉是好看的，但感觉不够想喝。年轻一点，但不要太网红。”
这句话背后可能包含多个真实诉求：
- 产品卖点没有被打出来；
- 活动利益点不明显；
- 视觉有氛围，但转化力不足；
- 客户希望年轻化，但又担心品牌质感被拉低；
- 团队不能只改文案，还要改设计、脚本、提案逻辑和客户回复口径。

所以这个 Agent 的价值不是“总结客户说了什么”，而是进一步回答：
客户到底想要什么？
哪些地方必须改？
哪些地方需要反问客户？
能不能先生成几个小方向，让客户确认到底想要哪种？

---
三、Agent 核心价值点
1. 从客户原话中提炼真实需求
客户说的往往不是最终需求，而是感受和判断。Agent 需要识别：
- 客户显性提出了什么；
- 客户真正担心什么；
- 客户没说出口但实际在意什么；
- 反馈中是否存在矛盾。
例如：
“品牌露出不够，但不要太硬广。”
这句话的核心矛盾是：
品牌要更明显，但内容不能像广告。

---
2. 从“反馈分析”升级到“执行方案”
普通 AI 只能分析客户是什么意思。
我们的 Agent 要进一步输出：
- 具体要改哪里；
- 谁来改；
- 优先改什么；
- 哪些需要向客户确认；
- 下一版方案怎么呈现；
- AE 如何专业回复客户。
- 是否需要生成几个小样给客户确认方向。
3. 降低多轮返工成本
广告项目的成本不只是创意生产，而是反复沟通、反复修改、反复理解客户意图。
Agent 的价值在于：
- 减少团队误读客户反馈；
- 减少无效返工；
- 提高下一轮方案命中率；
- 让客户反馈变成可追踪、可执行、可复用的团队资产。
4. 帮助新人 AE / 策划快速进入客户语境
资深广告人能听懂客户潜台词，但新人往往只能照字面理解。
Agent 可以把经验型判断标准化，让新人也能快速判断客户到底想要什么。
5. 沉淀客户偏好和修改逻辑
未来可扩展为客户偏好库：
某客户是否更在意品牌露出、调性安全、预算解释、领导审美、平台风险等。

---
四、核心用户
主要用户：
- 广告公司 AE / 项目执行
- 策划
- 媒介
- 设计
- 视频编导
- 品牌市场部
核心使用场景：
- 客户看完方案初稿后反馈；
- 客户会议结束后整理修改方向；
- 微信/邮件收到客户抽象意见；
- 内部需要将客户反馈同步给策划、媒介、设计、视频团队；
- 下一版方案提交前，需要明确修改逻辑和回复口径。

---
五、48 小时 MVP 范围
本次黑客松不做完整协作系统，只做一个最小闭环：
用户粘贴一段混乱客户反馈
Agent 自动整理信息
输出一张「甲方反馈行动卡」
并生成 2-3 个「方向确认小样」
MVP 不做
- 不接入真实企业微信 / 飞书；
- 不训练模型；
- 不做复杂知识库；
- 不处理真实客户数据；
- 不做完整项目管理系统。
MVP 要做
- 一键粘贴混乱反馈；
- 自动识别客户核心反馈；
- 萃取真实需求；
- 识别矛盾点和风险点；
- 生成分岗位行动建议；
- 生成客户回复话术；
- 生成 2-3 个方向确认小样；
- 用卡片式页面展示结果。

六、产品流程
Step 1：一键粘贴客户反馈
为了降低使用门槛，用户不需要逐项填写完整表单，只需要将客户反馈、微信聊天记录、邮件、会议纪要或方案批注直接粘贴进输入框。
产品交互采用：
一个主输入框 + 三个轻量标签

主输入框
用户粘贴客户反馈原文，支持：
- 微信聊天；
- 邮件；
- 会议纪要；
- 方案批注；
- 客户零散意见。
轻量标签
用户只需要简单选择：
1. 项目类型
例如：新品推广 / KOL 投放 / 品牌海报 / 短视频脚本 / 提案方案。
2. 当前阶段
例如：初稿反馈 / 二轮修改 / 执行前确认 / 上线前确认。
3. 输出对象
例如：给 AE / 给策划 / 给设计 / 给媒介 / 给客户回复。
4. 客户偏好/性格/爱好


示例输入
客户反馈：
整体视觉是好看的，但我们感觉“想喝”的感觉还不够强。现在更像一张氛围图，产品卖点没有被打出来。第二杯半价这个活动信息也有点弱。另外，希望年轻一点，但不要太网红、太花哨。
项目类型：新品推广
当前阶段：初稿反馈
输出对象：AE / 策划 / 设计 / 客户回复
Agent 会自动从非结构化信息中识别：
- 项目大致背景；
- 客户原话；
- 核心反馈点；
- 涉及岗位；
- 当前最需要推进的下一步。
如果信息不足，Agent 不直接强行生成结论，而是提示需要补充的问题，例如：
- 这是海报、短视频还是整套提案？
- 客户说“不够高级”主要指视觉、文案、达人还是整体调性？
- 是否已有上一版方案摘要？
- 本轮修改是要给客户确认方向，还是直接进入执行？

---
Step 2：甲方需求萃取
Agent 将客户反馈拆成以下字段：
1. 甲方原话
保留客户原始表达。
2. 一句话翻译
用广告团队能理解的话总结客户真实意思。
3. 显性需求
客户明确说出来的要求。
4. 隐性诉求
客户没有直接说，但实际在意的问题。
5. 核心矛盾
客户反馈中互相拉扯的地方。
例如：品牌要更明显，但不能太硬广。
6. 反馈类型
例如：品牌露出 / 创意调性 / 达人策略 / 预算逻辑 / 平台风险 / 内容传播力。
7. 优先级
分为：必须修改 / 建议优化 / 需要确认。
8. 风险点
如果理解错或改错，可能造成什么后果。
9. 需要反问客户的问题
哪些点不能直接猜，需要进一步确认。

需求确认
- 动态追问：基于已知信息逐轮提出最该问的问题，逼出清晰思路（用问题引导甲方，呼应原始构想）。
- 实时生成可点变体（杀手锏）：为同一需求当场生成 2–3 个视觉迥异、真实可点的网页原型，在页面内并排渲染。甲方点一个 + 吐一句槽。
- 从选择反推偏好：从"选了哪个 + 吐了什么槽"反推真实偏好——例如"选了数字优先版但嫌太刺激 → 真实要的是醒目但稳健"。证明 AI 是从行为反推，而非预设答案。
- 结构化需求规格：产品、目标用户、核心目标、设计原则、暂不优先项、待决策层确认项（缺口标红）。
- 五层需求图（标注缺口与冲突）：把一句话需求穿透成五层—— 
  - 上层：领导意图 / 政策合规 / 市场要求 / 业务流程
  - 下层：终端用户痛点
  - 每层标注状态（已明确 / 需注意 / 缺口），并显式标出哪两层在打架（如"市场要炫酷 vs 用户要安心"）。
- 风险红旗清单：合规红线、目标缺口、体验反噬、转化断点，各带风险等级。

---
Step 3：多角色 Agent 分析
可以设计为 5 个子 Agent：
1. 客户解读 Agent
负责理解客户原话背后的真实诉求。
2. 创意策略 Agent
负责判断内容方向、品牌关联和传播钩子怎么调整。
3. 执行拆解 Agent
负责把反馈拆成策划、媒介、设计、视频、AE 能执行的任务。
4. 风险识别 Agent
负责识别反馈矛盾、客户未说清楚的点、可能背锅的地方。
5. 沟通话术 Agent
负责生成可直接回复客户的专业话术。

六、核心产出物
最终页面输出一张「甲方反馈行动卡」，包含以下产出物。

---
产出物 1：甲方需求卡
内容包括：
- 客户原话；
- 一句话翻译；
- 显性需求；
- 隐性诉求；
- 核心矛盾；
- 反馈类型；
- 优先级；
- 风险点；
- 待确认问题。

---
产出物 2：方案修改清单
例如：
- 新增一页“产品卖点 / 品牌植入逻辑”；
- 调整主视觉信息层级；
- 优化活动利益点露出；
- 补充达人选择逻辑；
- 修改短视频脚本前 3 秒；
- 增加客户可能关心的执行说明。
产出物 3：客户回复话术
生成一段可以直接发给客户的回复，要求：
- 不卑微；
- 不过度承诺；
- 体现理解客户诉求；
- 说明下一版修改方向；
- 必要时提出需要确认的问题。
产出物 4：方向确认小样
这是本项目的关键新增产出物。
很多时候客户自己也说不清楚“高级一点”“自然一点”“年轻一点”到底长什么样。
与其团队继续猜，不如让 Agent 生成 2-3 个轻量方向小样，帮助 AE 反向确认客户偏好。
这个产出物的目的不是直接生成最终方案，而是生成几个低成本、轻量级的表达样本，用来和客户快速确认方向。
产出内容包括：
- 2-3 个不同修改方向；
- 每个方向的一句话策略；
- 每个方向的示例表达 / 小文案 / 脚本片段 / 原型；
- 适合解决客户哪一类反馈；
- 可能风险；
- 推荐优先级。
小样类型可以包括：
1. 轻度调整版
保留原方案结构，只微调表达和信息层级。
2. 策略强化版
不只是局部修改，而是重新强化产品、品牌或场景关系。
3. 创意升级版
在原方向基础上增加一个更明确的传播钩子或互动玩法。
AE 可以拿这些小样去问客户：
我们理解您想加强产品吸引力和活动信息，同时保持年轻质感。下面先提供三个轻量方向，您看哪一种更接近您想要的感觉，我们再基于确认方向展开下一版方案。

八、Demo 主案例｜大众版
项目背景
某连锁咖啡品牌计划推广夏季新品「白桃冰美式」，希望通过门店海报、小红书图文和短视频内容，吸引 18-30 岁年轻消费者到店尝鲜。
当前初稿方案主打“夏日清爽感”，视觉以粉色、冰块、白桃元素为主，文案方向偏轻松生活方式。
当前方案摘要
初稿标题：
把夏天喝成一口白桃味。
初稿内容方向：
用清爽画面和生活化文案表现新品的夏日氛围，画面中有产品杯、白桃、冰块和年轻人街边散步场景。活动信息放在页面底部：新品第二杯半价。
甲方反馈原话
整体视觉是好看的，夏天的感觉也有了，但是我们看下来觉得“想喝”的感觉还不够强。现在更像一张氛围图，产品本身的卖点没有被打出来。还有第二杯半价这个活动信息也有点弱，用户可能一眼看不到。
另外，我们希望年轻一点，但不要做得太网红、太花哨，还是要有一点品牌质感。你们能不能再调整一下，让它既有食欲感，又能把活动利益点说清楚？

---
九、Agent 输出示例
1. 一句话翻译
客户不是觉得画面不好看，而是担心广告“好看但不卖货”：产品吸引力、购买理由和活动利益点都不够突出。
2. 显性需求
- 产品本身要更有“想喝”的感觉；
- 新品卖点要更明确；
- “第二杯半价”活动信息要更突出；
- 视觉和文案要更年轻，但不能太网红、太廉价。
3. 隐性诉求
- 客户希望广告不只是有氛围，还要能促进到店购买；
- 客户担心年轻化表达过度，会伤害品牌质感；
- 客户需要下一版方案更明确地回答：用户为什么现在要买这杯新品？
4. 核心矛盾
客户既想要年轻化、清爽、有传播感，又不希望画面变得太花哨、太网红；既想要氛围感，又希望销售利益点足够清楚。
5. 反馈类型
产品卖点 / 视觉调性 / 活动信息 / 转化引导 / 品牌质感
6. 风险点
- 如果只加强活动信息，可能变成促销海报，削弱品牌质感；
- 如果只强化氛围，用户仍然不知道产品好喝在哪里；
- 如果年轻化过度，可能被客户认为“不像品牌”。

---
十、具体执行方案
策划改法
将核心策略从“夏日氛围表达”调整为“清爽口感 + 即刻尝鲜理由”。
下一版不只讲“夏天很好看”，而是要讲清楚：
- 这杯新品为什么适合夏天；
- 它的口感卖点是什么；
- 为什么用户现在值得买。
文案改法
原文案：
把夏天喝成一口白桃味。
建议优化为：
白桃香气 + 冰爽美式，一口把夏天降下来。
或者：
今天的降温计划：一杯白桃冰美式，第二杯半价。
设计改法
- 放大产品杯和冰爽质感；
- 增加白桃切面、冰块水珠、杯壁冷凝水等细节；
- 将“第二杯半价”从底部角落调整为右上角利益点标签；
- 整体色彩保持清爽，不使用过多贴纸、弹幕、网红字体。
短视频改法
原方向：
年轻人拿着咖啡在街边散步，整体偏生活氛围。
建议调整：
前 3 秒先给到产品特写：冰块、白桃、咖啡倒入、杯壁水珠；中段再切入年轻人饮用场景；结尾强调“新品尝鲜，第二杯半价”。
AE 对客户回复话术
收到，我们理解您这边主要希望下一版不只是保留夏日氛围感，而是进一步强化产品本身的吸引力和到店购买理由。下一版我们会把重点从单纯“好看清爽”调整为“清爽口感 + 明确利益点”，同时控制年轻化表达的尺度，避免过度网红化，保持品牌质感。

---

它的核心链路是：
混乱反馈 → 需求萃取 → 风险识别 → 分岗位执行 → 客户回复 → 方向小样确认
最终帮助广告团队减少误解、减少返工、提高下一轮方案通过率。
